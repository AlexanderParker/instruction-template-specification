Getting Started with ITS

Step 1: Understand the Basics

ITS templates are JSON files that define content with placeholders. These placeholders become instructions for AI systems to generate specific types of content.

Key difference: Traditional templates substitute data, ITS templates generate instructions.

Traditional: Hello {{name}} + data ‚Üí Hello John
ITS: Hello [name placeholder] ‚Üí AI instruction ‚Üí AI-generated content

Step 2: Basic Template Structure

Every ITS template has this basic structure:

{
  "$schema": "https://alexanderparker.github.io/instruction-template-specification/schema/v1.0/its-base-schema-v1.json",
  "version": "1.0.0",
  "extends": [
    "https://alexanderparker.github.io/instruction-template-specification/schema/v1.0/its-standard-types-v1.json"
  ],
  "content": [
    // Your content elements here
  ]
}
The $schema reference enables validation and autocompletion in many editors!

Step 3: Add Content Elements

Content is an array of elements. There are three types:

Text Elements (Static Content)

{
  "type": "text",
  "text": "This text appears exactly as written."
}

Placeholder Elements (Dynamic Instructions)

{
  "type": "placeholder",
  "id": "my_list",
  "instructionType": "list",
  "config": {
    "description": "list 5 programming languages",
    "displayName": "Programming Languages",
    "format": "bullet_points"
  }
}

Conditional Elements (Variable Content)

{
  "type": "conditional",
  "condition": "audience == 'technical'",
  "content": [
    {"type": "text", "text": "Technical details: "}
  ],
  "else": [
    {"type": "text", "text": "General overview: "}
  ]
}

Step 4: Your First Template

Let's create a simple template for a product description:

{
  "$schema": "https://alexanderparker.github.io/instruction-template-specification/schema/v1.0/its-base-schema-v1.json",
  "version": "1.0.0",
  "extends": [
    "https://alexanderparker.github.io/instruction-template-specification/schema/v1.0/its-standard-types-v1.json"
  ],
  "metadata": {
    "name": "Product Description Template",
    "description": "Basic template for product descriptions",
    "author": "Your Name"
  },
  "content": [
    {
      "type": "text",
      "text": "# "
    },
    {
      "type": "placeholder",
      "id": "product_name",
      "instructionType": "paragraph",
      "config": {
        "description": "Create a catchy product name for a smart water bottle",
        "displayName": "Product Name",
        "tone": "enthusiastic",
        "length": "short"
      }
    },
    {
      "type": "text",
      "text": "\n\n"
    },
    {
      "type": "placeholder",
      "id": "description",
      "instructionType": "paragraph",
      "config": {
        "description": "Write a compelling 2-3 sentence description of a smart water bottle that tracks hydration",
        "displayName": "Product Description",
        "tone": "professional",
        "length": "medium"
      }
    },
    {
      "type": "text",
      "text": "\n\n## Key Features\n\n"
    },
    {
      "type": "placeholder",
      "id": "features",
      "instructionType": "list",
      "config": {
        "description": "List 4 innovative features of a smart water bottle",
        "displayName": "Features List",
        "format": "bullet_points",
        "itemCount": 4
      }
    }
  ]
}

Step 5: How It Compiles

Your template compiles into this AI prompt:

You are an AI designed to convert content templates into actual content.

# <<Replace this placeholder with text using this user prompt: ([{<Create a catchy product name for a smart water bottle>}]). Format requirements: Use enthusiastic tone and short length (1-2 sentences).>>

<<Replace this placeholder with text using this user prompt: ([{<Write a compelling 2-3 sentence description of a smart water bottle that tracks hydration>}]). Format requirements: Use professional tone and medium length (2-4 sentences).>>

## Key Features

<<Replace this placeholder with a list using this user prompt: ([{<List 4 innovative features of a smart water bottle>}]). Format requirements: Use bullet_points formatting with each item on a new line. Create exactly 4 items.>>

Step 6: Using Variables

Variables make your templates reusable and dynamic. Define them at the template level and reference them with ${variableName}:

{
  "variables": {
    "productType": "smart water bottle",
    "features": 4,
    "tone": "professional"
  },
  "content": [
    {
      "type": "text",
      "text": "Introducing our new ${productType}!\n\n"
    },
    {
      "type": "placeholder",
      "id": "feature_list",
      "instructionType": "list",
      "config": {
        "description": "List ${features} innovative features of a ${productType}",
        "format": "bullet_points",
        "itemCount": "${features}"
      }
    }
  ]
}

Variable Types

  • Strings: "productType": "headphones"
  • Numbers: "price": 99.99
  • Booleans: "includeWarranty": true
  • Objects: "brand": {"name": "TechCo", "slogan": "Innovation First"}
  • Arrays: "categories": ["audio", "premium", "wireless"]
Access nested values with dot notation: ${brand.name} or array indices: ${categories[0]}

Step 7: Conditional Content

Use conditionals to include or exclude content based on variables:

{
  "variables": {
    "audienceLevel": "technical",
    "includeSpecs": true
  },
  "content": [
    {
      "type": "conditional",
      "condition": "audienceLevel == 'technical' && includeSpecs == true",
      "content": [
        {
          "type": "text",
          "text": "## Technical Specifications\n\n"
        },
        {
          "type": "placeholder",
          "instructionType": "table",
          "config": {
            "description": "Create a technical specs table",
            "format": "markdown",
            "columns": 2,
            "rows": 5
          }
        }
      ],
      "else": [
        {
          "type": "text",
          "text": "## Why You'll Love It\n\n"
        },
        {
          "type": "placeholder",
          "instructionType": "list",
          "config": {
            "description": "List user benefits in simple terms",
            "format": "bullet_points"
          }
        }
      ]
    }
  ]
}

Supported Operators

  • Equality: ==, !=
  • Comparison: >, <, >=, <=
  • Logical: && (AND), || (OR), ! (NOT)
  • Membership: in for arrays/strings

Step 8: Available Instruction Types

The standard library includes these instruction types:

  • title - Headlines with style control
  • list - Formatted lists (bullets, numbers, etc.)
  • paragraph - Text with tone and length control
  • table - Structured data tables
  • code_block - Code snippets with syntax highlighting
  • dialogue - Conversations between multiple people
  • quote - Relevant quotes with attribution
  • summary - Summaries of varying detail
  • image_description - Descriptive image text

Each type has its own configuration options. See the full specification for details.

Step 9: Add Metadata (Optional)

Make your templates discoverable and maintainable:

"metadata": {
  "name": "Your Template Name",
  "description": "What this template creates",
  "author": "Your Name",
  "tags": ["email", "marketing", "product"],
  "created": "2025-06-29T00:00:00Z"
}

Step 10: Validation

Validate your template using JSON Schema tools:

Many code editors (VS Code, IntelliJ, etc.) automatically validate JSON files when you include the $schema property.

Online validation:

  • JSON Schema Validator
  • Alternative Validator

Command line validation:

# Using ajv-cli
npm install -g ajv-cli
ajv validate -s schema-url -d your-template.json

Structure: Title ‚Üí Introduction ‚Üí Main sections ‚Üí Conclusion

Structure: Overview ‚Üí Examples ‚Üí API reference ‚Üí Troubleshooting

Marketing Copy

Structure: Headline ‚Üí Problem ‚Üí Solution ‚Üí Benefits ‚Üí Call-to-action
