üçä Citrus Fruits Example - Detailed Breakdown

Template JSON

{
  "$schema": "https://alexanderparker.github.io/instruction-template-specification/schema/v1.0/its-base-schema-v1.json",
  "version": "1.0.0",
  "content": [
    {
      "type": "text",
      "text": "Hey there is a list of fruits here:\n"
    },
    {
      "type": "placeholder",
      "id": "citrus_list",
      "instructionType": "list",
      "config": {
        "description": "list 5 different citrus fruits",
        "format": "bullet_points",
        "itemCount": 5
      }
    }
  ]
}

Compiled AI Prompt

You are an AI designed to convert content templates into actual content.

Hey there is a list of fruits here:

<<Replace this placeholder with a list using this user prompt: "list 5 different citrus fruits". Format requirements: Use bullet_points formatting with each item on a new line. Create exactly 5 items.>>

Key Learning Points:

  • Combines static text with dynamic placeholders
  • Uses the standard list instruction type
  • Configures specific formatting and item count
  • Demonstrates the basic compilation process

üìù Blog Post Template - Structure Analysis

This template demonstrates a complex content structure using multiple instruction types:

Template Structure

  • Title: paragraph type with enthusiastic tone, short length
  • Introduction: paragraph type with professional tone, medium length
  • Key Technologies: list type with numbered format, 6 items
  • Challenges Table: table type with markdown format, 3x4 dimensions
  • Expert Quote: quote type with attribution enabled
  • Conclusion: summary type with standard length
Usage Pattern:
Text ‚Üí Placeholder ‚Üí Text ‚Üí Placeholder ‚Üí Text...

This pattern creates a natural content flow where AI-generated sections are seamlessly integrated with static structure. Each placeholder clearly separates user content requests from system formatting requirements.

Best Practices Demonstrated:

  • Consistent heading structure using static text
  • Varied instruction types for content diversity
  • Appropriate tone selection for different sections
  • Descriptive placeholder IDs and display names

Creating Your Own Templates

Ready to create your own templates? Here are some tips:

Key Improvement: The new instruction format clearly separates user content requests (in quotes) from system formatting requirements, making it much easier for AI models to understand what content to generate vs how to format it.

Start Simple: Begin with basic text + list/paragraph combinations
Add Complexity: Gradually introduce tables, dialogues, and custom types
Test Compilation: Verify your templates compile to clear AI instructions
Use Metadata: Document your templates for future reference and sharing
